### Fork 2 — ZIP nested-path handling

`createZip` (main.js:328-429) currently handles flat filenames. Copilot is `.github/copilot-instructions.md`, Cursor (Fork 1-a) is `.cursor/rules/aipolicy.mdc`. Both need nested paths.

| | Option | Trade |
|--|--------|-------|
| 2-a **[recommended]** | Extend `createZip` to accept `/` in entry names; rely on unzip to create directories implicitly | ~0 LOC change to writer (filename is already a string). Test on mac/linux/windows unzippers. |
| 2-b | Flatten: ship files at root + a README explaining placement | Friction for adopters. Defeats "no install" promise. |
| 2-c | Add explicit directory entries to central directory | ZIP-spec-correct but unnecessary; modern unzippers infer dirs from filename. |

**Recommendation:** 2-a. Implementation is filename = `'.github/copilot-instructions.md'`; ZIP writer treats it as opaque string. Verification step in plan covers unzip cross-platform.

### Fork 3 — UI surfacing

| | Option | Trade |

# Spec — v1.3.0 Ecosystem Coverage

## Problem

aipolicy generates AI_POLICY.md + AGENTS.md + CLAUDE.md. The AI coding tool ecosystem has grown: Copilot (`.github/copilot-instructions.md`) and Cursor (`.cursor/rules/*.mdc`) both have project-specific instruction conventions that aipolicy currently ignores. Adopters who use these tools have to re-author rules manually.

## Approach

Add two new template generators emitting Copilot and Cursor files alongside the existing three. Content is reformatted (not copy-pasted) from the existing AGENTS.md generator to match each tool's documented convention. Both files participate in: preview tabs, ZIP download, static preset files for curl access. No new customize options — reuse existing `code_style`, `restricted_paths`, `test_reqs`, `review_reqs`, `disclosure`. Per-preset content varies via the same option set.

## Implementation

### Phase A — Templates (lib/templates.js) + UX de-risking

**0. De-risk tab-bar mobile UX before writing template code.** 30-second test: `make serve`, open in browser, narrow viewport to 320px, use DevTools to inject 2 dummy tab buttons into the existing `.tab-bar`. Verify horizontal scroll behavior or wrapping is acceptable. If broken, scope a Phase B CSS task; otherwise no additional work.

1. **Add `generateCopilot(opts)`**. Returns markdown for `.github/copilot-instructions.md`. Format follows Copilot's documented convention: plain markdown without frontmatter, single document. Content adapted from `generateAgents` but framed for Copilot's audience ("Instructions for GitHub Copilot working in this repository").

   **Options the template responds to** (mirror `generateAgents`):
   - `code_style` — emit a "Code Style" section if non-empty.
   - `restricted_paths` — emit a "Restricted Paths" section listing each path.
   - `test_reqs` — branches: `none` / `recommended` / `required-before-merge`.
   - `review_reqs` — branches: `none` / `significant-changes` / `all-ai-prs`.
   - `disclosure` — branches: `encouraged` / `required` (emits `Assisted-by` trailer guidance).
   - `ai_usage` — softens or strengthens the "scope" guidance.

   **Options the template ignores:** `claude_handling` (CLAUDE.md-specific), `ai_cicd`, `ai_code`, `training_optout` (these belong in AI_POLICY.md, not agent rules).

2. **Add `generateCursor(opts)`**. Returns markdown for `aipolicy.mdc` (target path: `.cursor/rules/aipolicy.mdc`). `.mdc` supports YAML frontmatter for Cursor rule metadata. Minimal frontmatter v1:
   ```yaml
   ---
   description: AI policy and agent rules for this project
   alwaysApply: true
   ---
   ```
   Include a short HTML comment after frontmatter explaining the user can flip `alwaysApply: false` to reduce per-conversation token cost if desired.

   **Options the template responds to:** same set as Copilot (`code_style`, `restricted_paths`, `test_reqs`, `review_reqs`, `disclosure`, `ai_usage`).

   **Options the template ignores:** same as Copilot (`claude_handling`, `ai_cicd`, `ai_code`, `training_optout`).

**Verification:** unit-style — generate output for all 3 presets, eyeball each, run through `markdownlint-cli2` to confirm no markdown errors. Frontmatter YAML in `.mdc` is markdown-compatible (markdownlint default config tolerates leading `---` blocks).

### Phase B — UI plumbing (index.html + scripts/main.js)

3. **`public/index.html`** — add two tab buttons + two textarea panels:
   ```html
   <button type="button" role="tab" id="tab-copilot" aria-selected="false" aria-controls="panel-copilot" tabindex="-1">copilot-instructions.md</button>
   <button type="button" role="tab" id="tab-cursor" aria-selected="false" aria-controls="panel-cursor" tabindex="-1">aipolicy.mdc</button>
   ```
   Same pattern as existing tabs. Add corresponding `<textarea>` panels in `.preview-pane`.

4. **`public/scripts/main.js`** — update five places:
   - `FILE_NAMES` constant: add `copilot: '.github/copilot-instructions.md'`, `cursor: '.cursor/rules/aipolicy.mdc'`
   - `getGeneratedFiles()`: add `copilot: generateCopilot(state.options)`, `cursor: generateCursor(state.options)`
   - `renderPreview()` `panels` object: add `copilot`, `cursor` entries
   - `renderTabs()` `tabMap` + `panelMap`: add `copilot`, `cursor`
   - `initTabs()` `tabKeys`: extend to `['policy', 'agents', 'claude', 'copilot', 'cursor']`
   - Imports: add `generateCopilot, generateCursor` from `lib/templates.js`

5. **`initDownload()` zipFiles**: append two entries with nested paths:
   ```js
   { name: '.github/copilot-instructions.md', content: getFileContent('copilot') },
   { name: '.cursor/rules/aipolicy.mdc', content: getFileContent('cursor') },
   ```

**Verification:** load `make serve` page in browser, click each preset, verify all 5 tabs render correct content for each preset. Test arrow-key tab navigation wraps correctly across 5 tabs.

### Phase C — ZIP writer cross-platform validation

6. **No code change to `createZip`**. Filename field is already a string-of-bytes; ZIP spec accepts `/` in filenames and most unzip implementations create directories implicitly.

7. **Add automated ZIP-extraction test to `make check`.** Create `scripts/test-zip-extraction.js`:
   - Import the templates module + a Node-port of `createZip` (small shim, since the browser version uses `TextEncoder`/`DataView`/`Blob`; Node has all three since v18).
   - Generate a representative ZIP using the actual code path (`standard` preset).
   - Extract via Node's built-in `zlib` parsing of the central directory, OR shell out to `unzip -l` and `unzip -d` (available on macOS + Linux runners).
   - Assert the extracted tree contains `.github/copilot-instructions.md` and `.cursor/rules/aipolicy.mdc` at the expected paths and with non-empty content.
   - Wire into `npm test` so `make check` runs it.

   Windows verification: CI matrix already runs on `ubuntu-latest` via the shared `ci-node.yml` workflow. If full Windows coverage is desired, expand the matrix to `windows-latest` for this single test (Info-ZIP `unzip` is available on GitHub-hosted Windows runners). Note: Windows Explorer's built-in ZIP extraction remains the long-tail risk; recommend documenting "we tested with Info-ZIP `unzip` and macOS Archive Utility; if extraction fails in your tool, please report."

If the automated test fails on any platform: implement Fork 2-c fallback (explicit directory entries in central directory, ~15 LOC addition to `createZip`). Don't implement preemptively.

### Phase D — Static presets + build scripts (scripts/)

8. **Single source of truth for file types.** Before touching build scripts, add an exported constant to `lib/templates.js`:
   ```js
   export const FILE_TYPES = [
     { key: 'policy',  path: 'AI_POLICY.md',                  generator: generateAiPolicy },
     { key: 'agents',  path: 'AGENTS.md',                     generator: generateAgents },
     { key: 'claude',  path: 'CLAUDE.md',                     generator: generateClaude },
     { key: 'copilot', path: '.github/copilot-instructions.md', generator: generateCopilot },
     { key: 'cursor',  path: '.cursor/rules/aipolicy.mdc',    generator: generateCursor },
   ];
   ```
   Build scripts iterate over this; no hardcoded counts.

9. **`scripts/generate-presets.js`** — emit all `FILE_TYPES` into each preset directory under `presets/`. **Nest the paths** (mirror real placement): `presets/standard/.github/copilot-instructions.md`, `presets/standard/.cursor/rules/aipolicy.mdc`. This preserves the "exact file at exact path" promise of the curl flow.

10. **`scripts/test-presets.js`** — derive expected count from `FILE_TYPES.length × preset_combos`. No hardcoded number.

11. **`scripts/validate-presets.js`** — iterate over `FILE_TYPES` for each preset; for each file, read disk content and compare to `generator(PRESETS[preset])`. Fail with the specific filename that mismatched, not a count. Add a separate check that no *extra* files exist under each preset dir.

12. **`make generate`** — regenerate all preset files; commit the new tree (including the new `.github/` and `.cursor/` subdirectories under each preset).

**Verification:** `make check` passes. If a file type is added/removed in `FILE_TYPES`, no test count needs updating.

### Phase E — Docs + meta

15. **`public/index.html`** — update the meta description and og:description (current "Create AI_POLICY.md, AGENTS.md, and CLAUDE.md" is now incomplete). New description: "Create AI policy and AI tool config files for your repository. Supports AI_POLICY.md, AGENTS.md, CLAUDE.md, Copilot, and Cursor."

## Validation (capability hypothesis)

This release is **capability-driven, not user-validated**. No adopter explicitly asked for Copilot/Cursor support; the backlog item is internal-sourced. The release ships as a hypothesis: "adopters of aipolicy who also use Copilot/Cursor will value pre-filled instruction files."

**Success signal (90 days post-v1.3.0):**
- Cloudflare analytics show ≥5% of `/presets/*` curl-traffic hitting the new file paths (`copilot-instructions.md` or `aipolicy.mdc`), OR
- ZIP download volume sustained or increased (current "Download All" flow now ships 5 files; no proxy for which files users actually use, but ZIP volume = continued tool relevance).

**Action on failure to validate (90 days, both signals miss):**
- Pause v1.3.x ecosystem expansion (no windsurf, no aider, no continue).
- Reframe the tool's positioning before adding more file types.
- Surface this in the next `/pb-review-comprehensive`.

**Action on validation (90 days, both signals hit):**
- Proceed with v1.3.1 expansion (next tool, picked by stability of upstream convention).
